Why this tool exists

Built for practical protocol learning, not textbook screenshots.

In real FAT and automation labs, engineers often need a fast bridge between IEC 61850 devices and familiar visualization layers. ARServer keeps the flow visible: discover the IED, inspect the signal, bind the Modbus address, and optionally publish to MQTT.

IED Explorer

Add relay endpoints, discover logical nodes, monitor live values, and keep the IEC reference visible for troubleshooting.

Modbus Server

Expose selected signals as input registers or discrete inputs for DCS, SCADA, gateway testing, and FUXA dashboards.

MQTT Publisher

Route checked signals into broker topics with quality and state context for web HMI and IoT learning workflows.

Gateway topology

One bridge, two practical outputs.

The design is intentionally simple: IEC 61850 remains the source of truth, while ARServer publishes clean representations to both legacy control systems and modern web dashboards.

Field truth

A gateway is useful only when quality, mapping, and diagnostics stay visible.

Signal without quality is dangerous. ARServer keeps value, quality, status, and IEC reference visible so a Boolean does not become a blind Boolean.

Mapping must be inspectable. Modbus addresses and MQTT topic routes need to be readable by humans before they become maintainable by teams.

Learning tools should be honest. This project is designed for lab, FAT, and engineering exploration. Production deployment still needs proper cybersecurity, redundancy, and validation strategy.

How to use the software

Install ARServer, add the runtime pieces, then route IEC values.

ARServer is a Windows gateway application. Real IEC 61850 communication needs the libiec61850 .NET/native DLLs beside ArServer.exe. MQTT output needs an external broker such as Mosquitto, EMQX, or HiveMQ.

  1. 1 Download the Windows installer from GitHub Releases.
  2. 2

    For real IED testing, copy the libiec61850 runtime DLLs beside ArServer.exe: iec61850dotnet.dll and iec61850.dll.

  3. 3 Start an MQTT broker if MQTT output is enabled.
  4. 4 Add an IED, discover signals, select rows for Modbus and MQTT, then start runtime.
